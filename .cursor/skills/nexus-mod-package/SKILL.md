# Nexus mod package

## Quick start

1. Read [modupload.md](../../../modupload.md) for current shipped versions and full checklist.
2. Run the pack script (default = full Nexus pack):

```bash
.cursor/skills/nexus-mod-package/scripts/pack-nexus.sh
```

   Warns if any repo Lua file exceeds Lua's **200 local variable per-function limit** (UE4SS load failure). Uses `luac -p` when installed, else static analysis.

3. Update `~/Downloads/ModUploads/G1R_IndoorNight_NexusPack/00_README_INSTALLATION.txt` — version header, `build=` verify string, Part A + Part B. Re-zip if README changed.
4. **Write the Nexus changelog** (agent step — see below). Not generated by scripts.
5. Run pre-upload checklist (below).

## Pack types

| Type | Flag | Default zip |
|------|------|-------------|
| Full (Lua + MaxPerf config) | `--full` | `G1R_IndoorNight_NexusPack.zip` |
| Scripts only | `--scripts-only` | `G1R_IndoorNight.zip` |
| Config only | `--config-only` | `G1R_IndoorNight_MaxPerf_v<N>.zip` |

**Default release = full pack.** Players expect Lua mod and MaxPerf config together unless the user says otherwise.

## Include / exclude

**Lua (exactly five files):** `main.lua`, `config.lua`, `indoornight_control.lua`, `indoornight_brightness.lua`, `indoornight_reload.lua`

**Never in upload zips:** `check-night-feedback.sh`, `tools/`, `docs/`, `install.sh`, profile variants (`Engine.ini.streaming-*`, `switch-g1r-profile.sh`), repo markdown, logs.

**Config sources (repo → pack):**

| Pack | Repo |
|------|------|
| `Config_MaxPerf/.../Engine.ini` | `Config/ProfilePack/.../Engine.ini.maxperf` |
| `Config_MaxPerf/.../GameUserSettings.ini` | `.../GameUserSettings.ini.maxperf` |
| `Config_MaxPerf/.../Scalability.ini` | `.../Scalability.ini` |

Use `.maxperf` sources — not live CrossOver `Engine.ini`.

## Versions in sync

Before upload, align:

- `Scripts/main.lua` → `MOD_BUILD`
- Staging `00_README_INSTALLATION.txt` header + verify line
- `Engine.ini.maxperf` header (e.g. v13)
- Nexus changelog copy-paste (agent-written)
- `modupload.md` → "Current shipped" section

## Nexus changelog (agent writes this)

**Required deliverable** after every pack. Plain text, copy-paste ready for Nexus. **Do not** put changelog generation in `pack-nexus.sh` or other scripts.

### Format

```
<MOD_BUILD>
- <change>
- Fixed: <bugfix>
```

- **Line 1:** exact `MOD_BUILD` string from `Scripts/main.lua` (e.g. `v3.6.2-outdoorpoll`)
- **Bullets:** `- ` prefix, one player-visible change per line
- **Tone:** human, concise — no raw diffs, no file paths unless helpful (`Scripts/config.lua` ok)
- **Fixes:** prefix with `Fixed:`
- **Config changes:** fold into bullets when shipped in the same pack (e.g. `MaxPerf Engine.ini v13: …`)
- **Omit** install instructions, markdown headers, bold, verify strings — Nexus page handles that separately

### Example (target style)

```
v3.6.1-configlua
- User settings moved to Scripts/config.lua
- Fixed: Indoor night too dark
- Fixed: Leaving indoors feels slow to brighten up
```

### How to write it

1. Read current `MOD_BUILD` and `Engine.ini.maxperf` header version.
2. Compare vs last release — unzip previous `G1R_IndoorNight_NexusPack.zip` or read `modupload.md` "Current shipped":
   ```bash
   REPO="$(git rev-parse --show-toplevel)"
   PACK=~/Downloads/ModUploads/G1R_IndoorNight_NexusPack
   diff -u "$PACK/Config_MaxPerf/Local/G1R/Saved/Config/Windows/Engine.ini" \
           "$REPO/Config/ProfilePack/Local/G1R/Saved/Config/Windows/Engine.ini.maxperf"
   diff -q "$PACK/G1R_IndoorNight/Scripts/"*.lua "$REPO/Scripts/"*.lua
   git log --oneline <last-shipped-commit>..HEAD -- Scripts/ Config/ProfilePack/
   ```
3. Translate diffs/commits into player-facing bullets — what changed for them, not implementation detail.
4. **Output the changelog in your response** in a fenced code block so the user can copy-paste directly.
5. Update `modupload.md` "Current shipped" if the release is final.

## Pre-upload checklist

- [ ] `MOD_BUILD` matches README and changelog title line
- [ ] Five Lua files only under `G1R_IndoorNight/Scripts/`
- [ ] Lua local limit: each file ≤200 locals per function (`count-lua-locals.py`; pack script warns automatically)
- [ ] Config from `.maxperf` sources
- [ ] Full pack zip ≈ 18 file entries (dirs + 5 lua + 3 ini + README); no `.DS_Store`
- [ ] Changelog delivered in response (human, copy-paste format above)
